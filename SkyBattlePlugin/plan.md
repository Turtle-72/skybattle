# Sky Battle — план следующих этапов

## Фаза 2: Кастомный лут в сундуках

### Цель
Реализовать систему случайной генерации лута в сундуках на карте, аналогично MCC Island Sky Battle — игроки находят сундуки и получают оружие, блоки, еду и утилиты.

### Архитектура

```
net.skybattle.plugin.loot/
├── LootManager.java          — регистрация таблиц лута, заполнение сундуков
├── LootTable.java            — именованная таблица с весами и предметами
├── LootEntry.java            — одна запись (Material, min/max кол-во, шанс, enchantments)
└── ChestPopulateListener.java — обнаружение/регенерация сундуков при старте раунда
```

### Шаги реализации

1. **config/loot.yml** — описать таблицы лута:
   - `common` — базовые блоки (камень, дуб, стекло)
   - `weapons` — мечи, луки, стрелы с случайными зачарованиями
   - `tools` — кирки, топоры
   - `consumables` — еда, зелья, жемчуг края
   - `rare` — TNT, ведра лавы/воды, эндер-перлы

2. **LootTable** — парсинг YAML:
   ```yaml
   weapons:
     - material: IRON_SWORD
       min-amount: 1
       max-amount: 1
       weight: 30
       enchantments:
         SHARPNESS: 1-2
     - material: BOW
       weight: 20
   ```

3. **LootManager.populateChest(Location)** — заполнить один сундук 3–5 предметами из смешанных таблиц.

4. **Привязка к карте** — в `config.yml` секция `chests`:
   - Список координат сундуков (`world,x,y,z`)
   - Или авто-сканирование региона при загрузке арены

5. **Интеграция с GameManager**:
   - При переходе `STARTING → IN_GAME`: очистить и заполнить все сундуки
   - При `ENDED`: заблокировать повторное открытие до следующего раунда
   - Флаг `openedChests` — каждый сундук лутается один раз за раунд (как в MCC)

6. **Защита от повторного лута** — `PlayerInteractEvent`:
   - Если сундук уже был открыт в этом раунде — отменить или показать пустой инвентарь
   - Маркировать через PersistentDataContainer плагина

### Тестирование
- [ ] Сундуки заполняются при старте раунда
- [ ] Повторное открытие того же сундука не даёт лут
- [ ] Веса таблиц работают корректно (статистика на 100+ генераций)

## Фаза 3: Стартовые блоки игрокам

### Цель
При начале боя каждый игрок получает набор стартовых блоков для строительства острова/моста — ключевая механика Sky Battle.

### Архитектура

```
net.skybattle.plugin.kit/
├── StarterKitManager.java    — выдача и настройка стартового набора
└── StarterKit.java           — immutable-описание набора (список ItemStack)
```

### Шаги реализации

1. **config/kits.yml** — конфигурация стартового набора:
   ```yaml
   default-kit:
     blocks:
       - material: COBBLESTONE
         amount: 32
       - material: OAK_PLANKS
         amount: 16
       - material: SCAFFOLDING
         amount: 8
     tools:
       - material: STONE_PICKAXE
         amount: 1
   ```

2. **StarterKitManager.giveKit(Player)** — вызывается из `GameManager.beginInGamePhase()`:
   - Очистить инвентарь (уже есть)
   - Выдать блоки в хотбар (слоты 0–4)
   - Инструменты — в слот 8
   - Сообщение игроку: «Вы получили стартовый набор!»

3. **Ограничения во время игры**:
   - Запрет крафта (`PrepareItemCraftEvent`) — только лут + стартовые блоки
   - Опционально: лимит блоков (счётчик поставленных блоков на игрока)

4. **Балансировка** (итерация):
   - Настраиваемые пресеты: `solo`, `teams`, `ranked`
   - Возможность выдачи разных наборов через permissions (`skybattle.kit.builder`)

5. **Интеграция с BorderManager**:
   - Стартовые блоки выдаются до телепорта на спавн
   - Телепорт на персональные стартовые платформы (будущая фаза — `ArenaManager`)

### Тестирование
- [ ] Каждый участник получает одинаковый набор при старте
- [ ] Набор не выдаётся в лобби (WAITING/STARTING)
- [ ] При смерти набор не восстанавливается
- [ ] Крафт заблокирован во время IN_GAME

## Фаза 4 (дальнейшее развитие)

| Компонент | Описание |
|-----------|----------|
| `ArenaManager` | Загрузка арен, стартовые платформы, точки спавна |
| `TeamManager` | Командный режим (2v2, 4v4) |
| `StatsManager` | SQLite/MySQL — победы, убийства, статистика |
| `ScoreboardManager` | BossBar/Scoreboard — живые игроки, таймер границы |
| `SpectatorManager` | Режим наблюдателя после смерти с телепортом |
| PlaceholderAPI | `%skybattle_state%`, `%skybattle_alive%` |

---

## Приоритет задач

1. **StarterKitManager** — быстрый win, минимальная зависимость
2. **LootManager + loot.yml** — core gameplay loop
3. **ArenaManager** — когда карта будет готова
4. **Stats / Scoreboard** — polish
